# Purpose

This document defines the visual, interaction and experience principles of Veriq.

It is the single source of truth for every screen, component, animation and interaction.

Every UI implementation, design mockup and generated screen must follow this design system.

The objective is consistency.

The objective is trust.

The objective is clarity.

---

# Design Philosophy

## Intent

Veriq should never feel like traditional educational software.

Veriq should feel calm, intelligent and trustworthy.

The platform should reduce cognitive load rather than increase information density.

Users should feel guided rather than overwhelmed.

---

## Design Statement

Veriq is an AI-native Educational Decision Intelligence platform.

It is not an analytics dashboard.

It is not a reporting tool.

It is not a chatbot.

It is an intelligent workspace where AI continuously helps school leaders understand what matters.

---

# Emotional Journey

Every screen should create the following emotional progression.

```
Calm

↓

Understanding

↓

Confidence

↓

Action

↓

Relief
```

Users should never feel lost.

Users should never feel overwhelmed.

Users should always understand what requires attention.

---

# Design Principles

Every screen should follow these principles.

---

## Principle One

Clarity over complexity.

Remove anything that does not improve understanding.

---

## Principle Two

Understanding before metrics.

AI explains first.

Metrics support the explanation.

---

## Principle Three

Evidence before recommendation.

Every recommendation should be traceable to visible evidence.

---

## Principle Four

One dominant insight.

Every page should immediately communicate the most important thing.

---

## Principle Five

Whitespace is functionality.

Empty space helps users think.

Never fill space unnecessarily.

---

## Principle Six

Beauty creates trust.

Good typography, spacing and hierarchy increase confidence.

Visual quality is part of usability.

---

## Principle Seven

Consistency wins.

The same interaction should always behave the same way.

# AI-Native Philosophy

Beacon is not another feature.

Beacon is the product.

Every important workflow begins with Beacon.

Every important workflow ends with Beacon.

The interface should constantly communicate that intelligence is present.

Beacon should appear naturally across the experience.

Never isolate intelligence.

---

# The Attention Model

Every page should guide the eye naturally.

The reading order should be:

```
Greeting

↓

Beacon Brief

↓

Primary Insight

↓

Evidence

↓

Recommended Action

↓

Supporting Metrics

↓

Historical Detail
```

If the eye moves somewhere else first,

the hierarchy is incorrect.

---

# Information Density

Veriq is intentionally sparse.

Only the most important information should be visible.

Additional information should appear progressively.

Avoid overwhelming the user with:

too many charts

too many tables

too many KPIs

too many colours

too many actions

---

# The One Question Rule

Every primary screen answers one question.

School Pulse

"What deserves my attention today?"

Early Intervention

"What is beginning to go wrong?"

Beacon

"Why has Veriq reached this conclusion?"

Decision Brief

"What should happen next?"

Reports

"What changed?"

Settings

"How should Veriq work for my school?"

---

# The 70 / 20 / 10 Rule

Every page should approximately communicate:

70%

Understanding

20%

Evidence

10%

Metrics

Do not reverse this relationship.

---

# Beacon First

Beacon always appears before metrics.

Beacon should always explain:

what happened

why it matters

what should happen next

Metrics exist to support Beacon.

---

# Trust

Users should immediately know:

why a recommendation exists

where it came from

how confident Veriq is

what should happen next

Trust is created through transparency.

Never hide evidence.

---

# Decision Intelligence

Veriq should always move users toward action.

Every important screen should answer:

What should I do next?

Every recommendation should naturally lead to:

Decision Brief

Owner

Due Date

Success Metric

---

# Simplicity

Whenever two designs communicate the same information,

choose the simpler design.

Remove unnecessary cards.

Remove unnecessary charts.

Remove unnecessary decoration.

---

# Final Principle

Every design decision should support one mission.

Help educational leaders make better decisions before problems become results.

If a design does not support that mission,

it should be redesigned.
